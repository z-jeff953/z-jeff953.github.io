编译原理-01-实现一个简单的编译器

使用 TypeScript / Python / Golang 实现一个简单的 Lisp 部分语法的编译器

1 认识编译器

1.1 编译器的定义/作用

编译器就是将“一种语言（通常为高级语言）”翻译为“另一种语言（通常为低级语言）”的程序

所以在这里我们将要实现的是将 Lisp 语言翻译成 TypeScript / Python / Golang

1.2 常见的编译器

常见的编译器是讲编程语言翻译成及机器码, 在这个过程中通常要经历中间阶段

中间阶段的引入使得编译器的架构更加模块分离, 能在编译器编写团队之间实现更好的分工, 也能讲不通阶段的中间代码赋予不同的意义

1.2.1 C语言

C语言的编译器经历了四个阶段：预处理，编译，汇编，链接

预处理：C语言 -> C语言(宏定义展开、头文件展开、条件编译)

编译：检查语法，C语言 -> 汇编语言

汇编: 汇编语言 -> 01机器码

链接：找到依赖的库文件，将目标文件链接为可执行程序

1.2.2 Java

Java程序从源文件创建到程序运行要经过两大步骤：

1、Java文件会由编译器编译成class文件（字节码文件），会经过编译原理简单过程的前三步(词法分析 —> 语法分析 —> 语义分析和中间代码生成 —> 优化 —> 目标代码生成)；

.Java -> .Class 在这个过程中会压缩文件体积，解析部分注解

2、字节码由java虚拟机解释运行，解释执行即为目标代码生成并执行。因为java程序既要编译的同时也要经过JVM的解释运行

.Class -> Java汇编代码 将 Class 文件 检查语法，执行，编译成中间代码

1.3 编译器的工作阶段

大多数编译器可以分成三个阶段：解析，转换以及代码生成

  1. 解析（Parsing）是将最初原始的代码转换为一种更加抽象的表示（译者注：即AST）。
  2. 转换（Transformation）将对这个抽象的表示做一些处理，让它能做到编译器期望它做到的事情。
  3. 代码（Code Generation）生成接收处理之后的代码表示，然后把它转换成新的代码。

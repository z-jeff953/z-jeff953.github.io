Database( PostgreSQL ) - 01 开始数据库 , 使用DDL

数据库就是方便编程的excel, 他为了方便编程牺牲了很多展示上的优化, 效率高, 编程方便, 主要就是存数据, 并约定一系列约束和操作范式, 提高效率, 减少错误和冗余. 开始数据库将会介绍最基本的数据库使用方法和数据定义语言的使用.

数据库根据涉及可以大致划分为关系型和非关系型, 关系型中限制更多, 但是表示数据的关系更好, 出现错误冲突的几率更小, 非关系型主要是因为有的数据其实准确性不是太重要, 尤其是大数据时代很多数据都是没啥用的, 所以非关系型就是习生规范和范式, 提高读写效率的数据库, 关系型更复杂, 概念比较多, 需要系统学习(主要是概念和思想, 语句看文档) , 非关系型形态各异, 遇到了再看文档可以触类旁通.

1. UESTC数据库学习目标

1.1 学习内容

  1. 掌握数据库系统基本概念、数据模型原理、数据库编程语言、数据库系统管理、NoSQL数据库技术原理等基础知识。
  2. 掌握数据库建模设计技术，能够给出数据库的优化设计方案，培养数据库设计能力。
  3. 掌握数据库SQL语言和数据库管理技术，能够对实现结果给出合理性解释，培养数据库操作访问与数据库系统管理能力。
  4. 掌握数据库应用编程技术及其开发工具的使用，培养数据库应用编程能力，同时培养解决复杂工程问题的数据库应用系统开发能力。

总的来说, 就是四个方面,

  1. 数据库相关概念 —— 关系数据模型 , 数据库的意义 , NoSQL的概念
  2. 数据库设计 —— 逻辑数据模型, 物理数据模型, 概念数据模型等, 如何设计数据库和数据表及其之间的关系
  3. 数据库操作 —— 如何写SQL语句
  4. 数据库编程 —— 使用 JDBC

1.3 SQL实战准备

数据库和SQL是两个概念, 数据库是Database, SQL是Structured Query Language, 结构化查询语言, 是一种编程语言, 他是有标准协会的, 其他数据库实现他的标准, DBMS是执行SQL语句的管理软件, 准确的说, 数据库就是指放数据的仓库 , DBMS是管理这些数据的系统 , SQL是告诉DBMS 进行什么操作的语句

PostgreSQL是一种特性非常齐全的自由软件的对象-关系型数据库管理系统（ORDBMS）

PostgreSQL 名字含有SQL, 俗称数据库, 实际是DBMS

通过SQL的学习我们可以了解数据库的概念和功能

在SQL中, 语句之间分割是分号, 同级别项目之间分割使用的是逗号

SQL语句总览

SQL 语言简洁，易学易用，核心功能只需9个动词

① DDL ( data manipulation language ）：CREATE、DROP、ALTER
② DML ( data definition language ）：SELECT、INSERT、UPDATE、DELETE
③ DCL（ data control language ）：GRANT、REVOKE

1. 数据定义语言

（Data Definition Language，DDL）

SQL语言中用于创建或删除表。也可以定义索引（键），规定表之间间的约束。

① CREATE DATABASE - 创建新数据库
② ALTER DATABASE - 修改数据库
③ CREATE TABLE - 创建新表
④ ALTER TABLE - 变更（改变）数据库表
⑤ DROP TABLE - 删除表
⑥ CREATE INDEX - 创建索引（搜索键）
⑦ DROP INDEX - 删除索引

2. 数据操纵语言

（Data Manipulation Language，DML）

也有的把SELECT单独分出去叫做 数据查询语言 DQL ( Data Query Language ）

SQL语言中用于查询和增删修改表数据的语句

① SELECT - 从数据库表中获取数据
② UPDATE - 更新数据库表中的数据
③ DELETE - 从数据库表中删除数据
④ INSERT - 向数据库表中插入数据

3. 数据控制语言

（Data Control Language，DCL）

SQL语言中用于对数据访问权进行控制的语句。

① GRANT – 授予用户对数据对象的权限
② DENY – 拒绝授予用户对数据对象的权限
③ REVOKE – 撤消用户对数据对象的权限

4. 事务处理语言

（Transaction Process Language，TPL）

SQL语言中用于数据库内部事务处理的语句。

① BEGIN TRANSACTION – 开始事务
② COMMIT – 提交事务
③ ROLLBACK – 回滚事务

5. 游标控制语言

（Cursor Control Language，CCL）

SQL语言中用于数据库游标操作的语句。

① DECLARE CURSOR – 定义游标
② FETCH INTO – 提交游标数据
③ CLOSE CURSOR– 关闭游标

4. 数据库索引 ( Index )

4.1 认识索引

索引是数据库快的原因之一.

索引（Index）是一种按照关系表中指定列的取值顺序组织元组数据存储的数据结构，使用它可以加快表中数据的查询访问

当然使用索引也不是百利无害的, 不能说既然索引快就把所有列的数据都设置为索引

索引优点：

  1. 提高数据检索速度
  2. 可快速连接关联表
  3. 减少分组和排序时间

就是查的快

索引开销：

  1. 创建和维护索引都需要较大时间开销
  2. 索引会占用额外物理存储空间
  3. 数据操纵因维护索引带来系统性能开销

就是创建索引耗时间, 保存索引耗空间, 增删改数据的时候还要修改索引耗时间

什么情况下使用 / 避免使用索引？

索引必须是在 WHEHE 子句的过滤条件中使用非常频繁的列。

虽然索引的目的在于提高数据库的性能，但这里有几个情况需要避免使用索引。

使用索引时，需要考虑下列准则：

  • 索引不应该使用在较小的表上。
  • 索引不应该使用在有频繁的大批量的更新或插入操作的表上。
  • 索引不应该使用在含有大量的 NULL 值的列上。
  • 索引不应该使用在频繁操作的列上。

对于TABLE的列的创建修改删除 以及约束的创建修改 要重点掌握

其次, 创建后想要修改可以这样修改

操作 格式
增加列 ALTER TABLE <表名> ADD <新列名称><数据类型>
增加列的约束 ALTER TABLE <表名> ADD CONSTRAINT [完整性约束]
删除列 ALTER TABLE<表名> DROP COLUMN <列名>；
删除列的约束 ALTER TABLE<表名> DROP CONSTRAINT<完整性约束名>；
修改列名 ALTER TABLE <表名> RENAME <原列名> TO <新列名>；
(注意和修改表名对比 ALTER TABLE <表名> RENAME TO <新表名>; )
修改列类型 ALTER TABLE <表名> ALTER COLUMN <列名> TYPE<新的数据类型>；

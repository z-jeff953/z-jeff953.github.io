2. UML

统一建模语言(Unified Modeling Language，UML)是一种为面向对象系统的产品进行说明、可视化和编制文档的一种标准语言，是非专利的第三代建模和规约语言。UML是面向对象设计的建模工具，独立于任何具体程序设计语言。

UML用与面向对象分析和建模

2.1 类图(Class Diagram)

类图描述面向对象设计的静态结构, 用于表示类和它们之间的关系

核心是类的表示和类之间关系的表示

2.1.1 类的表示

1. 基本表示形式

类图标由三个部分组成：最上面是类名，中间是属性，下面是操作。

2. 属性和操作的基本格式
  • 属性规格说明格式：

可见性 属性名称：类型 [多重性] = 默认值 {特性字符串}

  • 操作规格说明格式：

可见性 操作名称（参数名称：类型）：返回值 {特性字符串}

3. 命名规范

类名在它的命名空间中唯一。类名以大写字母开头，省略多个单词之间的空格。

属性和操作在类的范围内必须无二义。属性和操作是以小写字母开头，后续单词的首字母大写，且同样省略空格。

抽象类和抽象操作用斜体表示。

4. 可见性类别
标志 可见性类型
+ Public
# Protected
- Private

2.1.2 类的关系表示

类的关系有关联, 泛化, 聚合, 组合, 依赖

表示方法 名称 含义
关联 动宾短语, 有联系就算, 有其他关系就不用关联关系
泛化 继承和接口关系, 由子类指向父类, 或者由实现指向接口
聚合 表示has a关系, 汽车聚合了轮胎和引擎, 但是那天汽车不用了, 轮胎也可以拿去干别的
也就是两个对象的生命周期是相互独立的
组合 表示contain a关系, 部分在整体之外就没有任何意义，整体拥有部分，部分的生命周期与整体式一样的。
依赖 一个类的实现需要另一个类的协助，比如人吃苹果, 需要任何苹果两个类

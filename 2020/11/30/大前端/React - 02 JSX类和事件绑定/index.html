JSX基础

什么是jsx

JSX其实是嵌入到JavaScript中的一种结构语法

  • JSX的顶层只能有一个根元素，所以我们很多时候会在外层包裹一个div原生（或者使用后面我们学习的Fragment）；

  • 为了方便阅读，我们通常在jsx的外层包裹一个小括号()，这样可以方便阅读，并且jsx可以进行换行书写；

  • JSX中的标签可以是单标签，也可以是双标签；

    • 注意：如果是单标签，必须以/>结尾；

jsx嵌入变量

  • 情况一：当变量是Number、String、Array类型时，可以直接显示
  • 情况二：当变量是null、undefined、Boolean类型时，内容为空；
    • 如果希望可以显示null、undefined、Boolean，那么需要转成字符串；利用空字符串拼接
    • 这样设置可以方便让一些元素不显示
  • 情况三：对象类型不能作为子元素（not valid as a React child）

this绑定

js的this指向问题:

谁调用 , 指向谁.

没有指定指向的时候指向window, 严格模式指向undefined

那render方法中看起来也没有更改指向啊,为什么要bind呢?

dom树上绑定事件是耗性能的, 是需要浏览器不停的监听的, 所以在react的虚拟dom中, 没有直接通过一个合成事件(SyntheticEvent)监听事件, 然后通过事件冒泡到document中执行对应的回调函数, 在回调的时候就会发生this指向失去的问题, 总结一下, 在react中使用了虚拟dom, 事件处理函数并没有直接的和dom结点绑定而是通过虚拟dom进行的调用, 所以会有失去this指向的情况发生

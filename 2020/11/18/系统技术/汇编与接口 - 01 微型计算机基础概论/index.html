1.1.2 冯 • 诺依曼计算机

Ⅰ 工作原理和结构特点

  1. 存储程序的工作方式

  2. 运算器为核心的结构特点

Ⅱ 特点和不足
① 特点
  1. 存储程序，共享数据，顺序执行；
  2. 属于顺序处理机，适于确定的算法和数值处理。
② 不足
  1. 与存储器间有大量数据交互，对总线要求很高；
  2. 执行顺序由程序决定，对大型复杂任务较困难；
  3. 以运算器为核心，处理效率较低；
  4. 由PC控制执行顺序，难以进行真正的并行处理。

1.1.3 非 冯 • 诺依曼计算机

Ⅰ主要特征
  • 并行性
Ⅱ 典型类型
① 数据流计算机结构 Dataflow Image Processing System

  • 采用数据驱动方式

    1. 程序的执行顺序不是由程序计数器控制，而是由指令间的数据流控制
    2. 当指令具有所需数据、且输出端没有数据时就可执行

② 哈佛结构 Harvard Architecture
  1. ==指令==和==数据==分别存放在两个独立的存储器模块中
  2. CPU与存储器间的指令和数据的传送分别采用两组独立的总线
  3. 可以在一个机器周期内同时获得指令操作码和操作数

1.1.4 微机系统的组成

Ⅰ 微处理器

集成了运算器单元、控制器单元、寄存器单元及内部总线，并具有CPU全部功能的超大规模集成电路芯片

Ⅱ 微型机

以微处理器为核心，配上内存、I/O接口、系统总线及电源等构成主机，连接输入/输出设备、外存设备等部件构成的硬件装置

Ⅲ 微机系统

为微型机硬件系统安装配置系统软件和应用软件后所构成的运行系统

微型机硬件系统 + 系统软件和应用软件 $\Longrightarrow$ 微机系统

总结

1.2.1 概述

  1. 按微处理器字长

    1,4,8,16,32,64位机

  2. 按结构

    单片机 , 多片机

  3. 按组装

    单板机 , 多板机

  4. 按外形

    台式微机 , 笔记本微机 , 掌上电脑

1.2.2 单片机介绍

Ⅰ 组成

集成了CPU、部分内存、部分I/O接口及总线等单元，具有微机基本功能的超大规模集成电路芯片

Ⅱ 特点
  1. 体积小
  2. 可靠性高
  3. 成本低
Ⅲ 应用领域
  1. 工业控制
  2. 智能仪器仪表
  3. 家用电器
  4. 其它各种嵌入式系统

1.2.3 台式微机(PC微机)介绍

Ⅰ 组成
  1. 主机箱 : 系统主板、 I/O接口卡、软/硬磁盘、光驱、电源等
  2. 外接设备 : 键盘、显示器、鼠标
  3. 软件 : 系统软件和应用软件
Ⅱ 特点
  1. 功能强
  2. 配置灵活
  3. 软件丰富
  4. 操作方便等
Ⅲ 应用领域
  1. 科学计算
  2. 事务处理
  3. 信息服务等许多领域

*1.3 微机系统发展概况

第一阶段 (1971 - 1973)
  • 低档4位或8位微处理器与微机
  • 系统结构与指令系统简单
  • 集成度低、运行速度慢
  • 机器语言或汇编语言编程
第二阶段 (1974 - 1978)
  • 中档8位微处理器与微机
  • 指令系统较丰富，具有典型计算机结构
  • 集成度提高到5000-9000管/片
  • 基本指令运行时间约1-2us
  • 出现高级语言编程与简单操作系统
第三阶段 (1978 - 1984)
  • 16位微处理器和微型计算机
  • 集成度和运算速度比第二代提高了一个数量级
  • 指令系统更加丰富，系统结构增加了多级中断机制、多寻址机制、段式存储器结构、硬件乘除部件
  • 支撑软件是操作系统
  • 外部设备种类增多
第四阶段 (1985 - 1991)
  • 32位微处理器和微型计算机
  • 微处理器芯片集成度达100万管/片
  • 运行速度超过25MIPS
  • 支持多用户多任务操作系统
第五阶段 (1991 - )
  • 高档32位、64位微处理器和微型计算机
  • 微处理器芯片集成度达2800万管/片以上
  • 时钟主频高达3.0 GHz以上
  • 支持多用户多任务操作系统

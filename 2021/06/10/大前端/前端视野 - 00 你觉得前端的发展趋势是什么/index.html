前端视野 - 00 你觉得前端的发展趋势是什么

面试的时候被问到这个问题, 整个人有点懵, 就提纲挈领的说了一下”前端领域一直在补落下的课”, 事实上我确实是这么认为的, 但是当时我没说清楚, 就说了工程化和编译时的问题, 所以现在整理一下.

不过后面仔细琢磨,应该是两个方面,

  1. 补和后端技术类似的工程化实践的工具的课
  2. 提高浏览器端的渲染性能

前端领域一直在补落下的课

首先这句话我们要对前端领域下一个较为准确的定义

前端是啥, 狭义的讲, 前端特指Web前端, 广义的讲, 所有的GUI都是前端, 游戏, Web , 移动端, 桌面端都是前端

A front-end developer is a programmer that codes the front end of a website. Typically, the front-end developer’s job is to convert website design files into raw HTML, JavaScript (JS) and/or CSS code. This includes the basic website design/layout, images, content, buttons, navigation and internal links. The end result is code that serves as the website’s front-end structure, which is used by a back-end developer to add business logics and connect databases and processes, among other processes.

A front-end developer is responsible for ensuring that a website’s visual front end is free of errors and looks exactly as designed. A front-end developer also ensures that a website has the same visibility across different computing and mobile Web browsers.

Similarly, in software applications, a front-end web developer creates the graphical user interface (GUI) that enables access to the software’s back-end features and capabilities.

但是我们常说的都是狭义前端的含义, 就是特指Web前端 , 比如招聘网站上都是这么说的, 如果要说所有的GUI , 那么就会称之为大前端

前端领域

1 为什么说前端都是在补课

一般提到这个问题，热评都是大前端，serverless，node，WASM之类的，但其实这些要么不是前端领域的内容，要么也是前端应该有但是没有的东西。

首先是不是前端领域的东西，大前端，Serverless，NodeJS都不是严格意义上的前端领域的内容。而是一个”前端程序员”的发展趋势。

原因如下

大前端是指Js，css，html写一切，有言道“一切能被JavaScript重写的都会被JavaScript重写“，包括NodeJS，在我看来这应该是Js以及其生态发展的结果，而不是前端的发展趋势，Js作为一个有国际标准组织，受众这么广泛，还挺易学的语言，他应用的领域越来越多并不奇怪。

Serverless更多的是运维的发展，云时代封装了硬件运维的部分，Serverless封装了软件和操作系统层面的运维部分暴露出了更加方便的API进行编程这很明显是运维的发展

WASM是把带来的隔离性，字节码又没很像JAVA，经过这么多年的发展，JS更像Java了

真正前端的发展是前端在工程化，编译时，类型检查方面，比如http import，微前端，tree shaking，ts这些都是编译和工程化的成果

我这么说不是因为我觉的前端可以不学flutter，RN，Node，Serverless这些东西，相反，我崇尚的是一个Web程序员应该全面的了解一个Web程序从前到后，从需求分析到系统运维整个生命周期的所有环节的内容，没有只会一个领域学不会另一个领域的说法，能够独当一面的程序员是刚需。

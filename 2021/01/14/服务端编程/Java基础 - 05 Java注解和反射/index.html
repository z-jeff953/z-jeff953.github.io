Java基础 - 05 Java注解和反射

Java注解和反射

1. 注解

当看到注解@Override的时候，我想到了JavaScript/Python的装饰器，对于动态语言，装饰器通过装饰器协议返回一个新的类/函数，对函数/类进行覆写，实际上是高阶函数，但对于Java而言，注解并不是装饰器, 他是“注解”。是一种标记，按照使用的目的，可以分为：1. 给人看的, 2.给编译器看的, 3.给解释器看的.

注解常见的作用有以下几种：

  1. 生成帮助文档。这是最常见的，也是 Java 最早提供的注解。常用的有 @see、@param 和 @return 等；
  2. 跟踪代码依赖性，实现替代配置文件功能。比较常见的是 Spring 2.5 开始的基于注解配置。作用就是减少配置。现在的框架基本都使用了这种配置来减少配置文件的数量；
  3. 在编译时进行格式检查。如把 @Override 注解放在方法前，如果这个方法并不是重写了父类方法，则编译时就能检查出。

无论是哪一种注解，本质上都一种数据类型，是一种接口类型。到 Java 8 为止 Java SE 提供了 11 个内置注解。其中有 5 个是基本注解，它们来自于 java.lang 包。有 6 个是元注解，它们来自于 java.lang.annotation 包，元注解用于定义注解。

1.1 基本注解

包括：@Override、@Deprecated、@SuppressWarnings、@SafeVarargs 和 @FunctionalInterface

1.1.1 @Override

用来指定方法重写的，只能修饰方法并且只能用于方法重写，不能修饰其它的元素。它可以强制一个子类必须重写父类方法或者实现接口的方法。

使用Override可以保证我们正确的覆写了父类的方法, 当使用父类没有的方法的时候就会报错.

1.1.2 @Deprecated

用来注解类、接口、成员方法和成员变量等，用于表示某个元素（类、方法等）已过时。当其他程序使用已过时的元素时，IDE和编译器将会给出警告。

Java 9 为 @Deprecated 注解增加了以下两个属性：

  1. forRemoval：该 boolean 类型的属性指定该 API 在将来是否会被删除。
  2. since：该 String 类型的属性指定该 API 从哪个版本被标记为过时。

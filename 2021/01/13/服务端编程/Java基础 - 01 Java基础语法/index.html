Java基础 - 01 基础语法

Java 基础语法

1. 认识Java

1.1 Java特点

纯面向对象, 跨平台, 强类型, 安全, 高效, 多线程, 分布式, 解释运行

Java有两种特殊的机制保证这些特性

  1. Java 的虚拟机机制(JVM) , 虚拟机机制保证Java程序的跨平台特性
  2. Java 的垃圾回收机制 , 回收机制保证Java程序更安全、更高效

1.2 Java 运行机制

Java是解释运行的, 但是不是完全的解释型语言, 可以编译成字节码再由JVM运行, 这样做既保证了跨平台性, 也保证了运行效率, 但从性质上讲, Java是解释型语言

1.3 认识JVM机制

Java虚拟机（Java Virtual Machine）是可运行Java字节码的虚拟计算机系统

使用Java语言编写的程序，实际上是运行在JVM之上，而不是运行在操作系统上

它有一个解释器组件，可以实现Java字节码和计算机操作系统之间的通信

3.1 的数据类型和运算符

3.1.1 数据类型及其范围

  1. 数据类型
    1. 简单类型
      1. 整数类型: byte、short、int、long
      2. 浮点类型: float、double

Java语言数据中的数值类型都是有符号(正负号)的

简单类型的变量被声明时，存储空间也同时被分配。该贮存空间只占用一个单一贮存单元。对简单类型变量访问则直接可以得到它的数据。

3.1.2 声明变量和C的区别

Java声明局部变量后必须初始化后使用, 未初始化可以声明, 但调用的时候会报错

浮点型

浮点型表示有小数部分的数值,float型和double型,浮点型数据默认为double型,因此要使用float型字面常量时，必须添加后缀F或f, 使用double型则不需要加F

boolean

在Java中boolean型的值不能与任何其他基本类型的值进行转换代替, 不能使用0和1表示false和true, booloean只有两种取值

3.1.4 变量类型

局部变量
  1. 局部变量声明在方法、构造方法或者语句块中；
  2. 局部变量在方法、构造方法、或者语句块被执行时创建，当执行完成后，变量将会被销毁；
  3. 访问修饰符不能用于局部变量；
  4. 局部变量只在声明它的方法、构造方法或者语句块中可见；
  5. 局部变量是在栈上分配的。
  6. 局部变量没有默认值，所以局部变量量被声明后，必须经过初始化，才可以使用。
实例变量
  1. 实例变量声明在一个类中，但在方法、构造方法和语句块之外；
  2. 当一个对象被实例化之后，每个实例变量的值就跟着确定；
  3. 实例变量在对象创建的时候创建，在对象被销毁的时候销毁；
类变量
  1. 类变量也称为静态变量，在类中以static关键字声明，但必须在方法、构造方法和语句块之外
  2. 无论一个类创建了多少个对象，类只拥有类变量的一份拷贝
  3. 静态变量在程序开始时创建，在程序结束时销毁
  4. 与实例变量具有相似的可见性。但为了对类的使用者可见，静态变量应声明public类型

3.3.3 日期类型

java.util 包提供了Date 类来封装当前的日期和时间。Date 类提供两个构造函数来实例化Date 对象。

  1. 第一个构造函数使用当前日期和时间来初始化对象Date()
  2. 第二个构造函数接收一个参数，该参数是从1970年1月1日起的毫秒数
    Date(long millisec)
SimpleDateFormat格式化日期

SimpleDateFormat是一个以语言环境敏感的方式来格式化和分析日期的类。
SimpleDateFormat允许选择任何自定义日期时间格式.

yyyy是公元年，MM是月份，dd是日期，HH:mm:ss是时、分、秒。

注意:有的格式大写，有的格式小写，例如MM是月份，mm是分；HH是24小时制，而hh是12小时制

使用printf和format进行方便的常用格式化方法
转换符 说明 示例
%tc 包括全部日期和时间信息 星期一九月18 14:21:20 CST 2017
%tF “年-月-日”格式 2017-09-18
%tD “月/日/年”格式 09/18/17
%tr “HH:MM:SS PM”格式（12时制） 02:25:51 下午
%tT “HH:MM:SS”格式（24时制） 14:28:16
%tR “HH:MM”格式（24时制） 14:28
